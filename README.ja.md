## 📦 ストレージシステム（Storage System）

[English](./README.md) | [日本語](./README.ja.md)

---

### 🔍 概要

**ストレージシステム（Storage System）** は、
**ファイル名だけでなく、OCR（光学文字認識）を利用してファイル内容からも検索できる**
ドキュメントストレージプラットフォームです。

多くのドキュメント管理システムでは、
「ファイルは保存されているが、中身は検索できない」という課題があります。
本システムは、その問題を解決することを目的としています。

---

### ❗ 解決したい課題

多くの組織では、以下のような問題を抱えています。

- 📁 PDF・画像・スキャン文書が大量に保存されている
- 🔎 検索はファイル名や簡単なメタ情報のみ
- 🧾 ドキュメント内の重要な情報が検索できない
- ⏱️ 必要な資料を探すのに時間がかかる

---

### 💡 解決策

ストレージシステムでは、OCR を用いて
**ファイル内部のテキストを抽出・インデックス化** することで、次のような検索を可能にします。

- 🔎 ファイル名による検索
- 📄 ファイル内容（テキスト）による検索
- ⚡ ドキュメント検索の高速化
- 🧠 保存されたデータの有効活用

---

### 🧱 システム構成

本システムは、責務を明確にするために複数のリポジトリに分割されています。

| リポジトリ                       | 役割                             |
| -------------------------------- | -------------------------------- |
| 🧩 **storage-system**            | システム全体の概要・ドキュメント |
| 🧠 **storage-domain**            | ビジネスロジック・ドメインルール |
| ⚙️ **storage-be**                | アプリケーション層・API          |
| 🛠️ **storage-content-processor** | OCR 処理・コンテンツ抽出ワーカー |
| 🎨 **storage-fe**                | ユーザーインターフェース         |

> クリーンアーキテクチャの考え方を採用し、
> ビジネスロジックをインフラから独立させています。

---

### 🔄 全体の処理フロー（概要）

1. 📤 ユーザーがファイルをアップロード
2. ⚙️ バックエンドがファイル情報を保存
3. 🛠️ OCR ワーカーがファイル内容を解析
4. 🧠 抽出されたテキストをインデックス化
5. 🔍 ファイル名・内容の両方で検索可能

---

### 🎯 目的

- ドキュメントの **「中身」を検索可能にする**
- ビジネスロジックと実装詳細を分離する
- 将来的な拡張（検索エンジン、処理方式の追加）を容易にする
